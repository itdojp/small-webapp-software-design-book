# STYLEGUIDE

本リポジトリの執筆・更新における最低限の規約です。可読性と変更容易性を優先します。

## 1. 対象範囲

- 本書の対象は **小〜中規模の TypeScript Web アプリ**（Web フロントエンドを含む）とする
- 次は原則として扱わない（必要なら別 Issue で合意する）
  - 特定フレームワーク固有の最適化（例: Next.js 固有のチューニング）
  - 大規模前提のテンプレ（マイクロサービス化、全域クリーンアーキテクチャ化）

## 2. 文章トーン

- 断定の根拠（前提条件、対象範囲、判断基準）を明確にする
- 価値（なぜ必要か）→手段（どうするか）の順で書く

## 3. 見出し粒度

- H1: 章タイトル（例: `01. 要件定義を設計入力にする`）
- H2: セクション（目的/前提/要点/演習…）
- H3: 必要最小限（増やし過ぎない）

## 4. 章ファイルの共通テンプレ

各章には以下のセクションを含める。

- 目的
- 前提/用語
- 要点（箇条書き）
- 演習（最小 1 個）
- よくある失敗
- チェックリスト
- 前後リンク（目次/前章/次章）

## 5. リンク規約

- 内部リンクは `relative_url` を基本とする（Project Pages の `baseurl` を考慮）
  - 例: `[目次]({{ '/chapters/' | relative_url }})`
- パスは原則として末尾 `/` を付ける（`permalink: pretty` 前提）

## 6. Front Matter 規約（章/付録）

章・付録は front matter に以下を含める。

- `title`
- `permalink`
- `show_nav: true`
- `prev` / `next`（`/chapters/.../` など Jekyll の `url` と一致する形式）

## 7. 著作権と引用

- 参照書籍・PDF本文の転載は禁止（リンクと最小限の引用に留める）
- 外部資料を引用する場合は、出典 URL と引用範囲を明示する
